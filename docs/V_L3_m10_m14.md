## 1. Layer Overview (from `mod.rs`)

L3 is the **keystone** layer — the HTTP hook server replacing all 8 bash hook scripts with sub-ms endpoints. Claude Code sends `POST http://localhost:8133/hooks/{event}`, ORAC processes and returns a decision.

| Module | File | Hook Events | Side Effects |
|--------|------|-------------|--------------|
| M10 | `m10_hook_server.rs` | — | Axum router, `OracState`, 16 query endpoints |
| M11 | `m11_session_hooks.rs` | `SessionStart`, `Stop` | Sphere register/deregister, POVM/RM hydration, ghost traces |
| M12 | `m12_tool_hooks.rs` | `PostToolUse`, `PreToolUse` | Memory record, task poll+claim, thermal gate, STDP dispatch |
| M13 | `m13_prompt_hooks.rs` | `UserPromptSubmit` | Field state injection, blackboard GC |
| M14 | `m14_permission_policy.rs` | `PermissionRequest` | Auto-approve/deny policy engine |

**Design invariants:**
- All handlers return within 1ms (local memory, no external I/O in hot path)
- Feature-gated: `#[cfg(feature = "api")]`
- Depends on: L1 (types, errors, config), L2 (IPC client)

## 6. Module Details

### 6.1 M11: Session Hooks (`m11_session_hooks.rs`, 403 lines)

**Handlers:**

#### `handle_session_start`
1. Generates unique pane ID (`{hostname}:{pid}:{uuid}`)
2. Registers sphere on PV2 (fire-and-forget POST `/sphere/{id}/register`)
3. Hydrates from POVM (`/hydrate`) + RM (`/search?q=discovery`) — consent-gated, parallel via `tokio::join!`
4. Tracks session in `OracState.sessions`
5. Registers in coupling network via `network.register()` (not raw HashMap — BUG-L4-001 fix)
6. Registers on blackboard: `pane_status` (Idle) + `agent_card` (4 capabilities)
7. Returns `[HABITAT] Hydrated: POVM N memories, N pathways | RM N discoveries`

#### `handle_stop`
1. Removes session from tracking
2. Fails active task on PV2 bus (fire-and-forget POST `/bus/fail/{task_id}`)
3. Marks sphere status "complete" on PV2 (skip for unknown sessions — BUG-060f)
4. Crystallizes to POVM (`/snapshots`) — consent + breaker gated
5. Crystallizes to RM (`/put`) — consent + breaker gated, **awaited with 3s timeout** (T6 fix: race condition on process exit)
6. Records blackboard task failure + marks pane Complete
7. Records ghost trace (sphere_id, persona, phase, tools, duration)
8. Deregisters from coupling network (BUG-L3-007: prevent unbounded growth)
9. Deregisters sphere on PV2 (skip for unknown sessions — BUG-060f)
10. Returns empty response

**Private helpers:** `parse_povm_hydration(data) -> (u64, u64)`, `parse_rm_count(data) -> usize`

**Tests:** 24 (parse_povm: 9, parse_rm: 7, message format: 2, session lifecycle: 6)

### 6.2 M12: Tool Hooks (`m12_tool_hooks.rs`, ~1,124 lines)

**Constants:** `POLL_EVERY_N = 5`

**Handlers:**

#### `handle_post_tool_use`
1. Finds session/pane from `HookEvent.session_id`
2. Increments `OracState.total_tool_calls` and `SessionTracker.total_tool_calls`
3. Records tool use as sphere memory (fire-and-forget/breaker POST `/sphere/{id}/memory`)
4. Updates sphere status to "working" (fire-and-forget/breaker POST `/sphere/{id}/status`)
5. Records blackboard pane status upsert (Working)
6. Records dispatch via semantic router (classify_tool, record_dispatch)
7. Records token accounting (input/output chars/4 estimate)
8. Records OTel trace span
9. Checks for `TASK_COMPLETE` in tool output → completes active task, records blackboard
10. Throttled task polling (1-in-5): calls `poll_route_and_claim`

#### `poll_route_and_claim` (private async)
1. GET `/bus/tasks` from PV2 (1000ms timeout)
2. Classifies each pending task via M20 semantic router (`classify_content`)
3. Routes to best-fit pane via M20 (`route` with field_state spheres + coupling)
4. If best pane matches this pane, attempts atomic claim POST `/bus/claim/{task_id}`
5. Fallback: claims first pending task if no task routes here (prevents livelock)
6. Returns system message with task description or empty

#### `handle_pre_tool_use`
1. Only gates write operations (`Write`, `Edit`, `Bash`)
2. Checks SYNTHEX thermal state via GET `/v3/thermal` (1000ms timeout)
3. Warns if temperature >30% over target — **advisory only, does NOT block**
4. Records breaker success/failure for synthex

**Private helpers (15):**
- `record_task_completion` — Blackboard task record + pane counter increment
- `is_write_operation` — `matches!(tool_name, "Write" | "Edit" | "Bash")`
- `truncate_string` — UTF-8 safe char-based truncation (BUG-L3-003)
- `find_pending_tasks` — Parse all pending tasks from bus response
- `find_first_pending_task` — Parse first pending task (test-only)
- `is_claim_successful` — Check claim response status
- `parse_thermal` — Parse SYNTHEX `{temperature, target}` response
- `find_session_pane` — Resolve session_id + pane_id from HookEvent
- `get_active_task` — Read active task ID for session
- `has_active_task` — Boolean check for active task
- `clear_active_task` — Clear active task for session
- `set_active_task` — Set active task + claim timestamp
- `get_task_claimed_ms` — Get claim epoch ms
- `increment_poll_counter` — Increment and return poll counter

**Structs:** `PendingTask { id: String, description: String }`

**Tests:** 62

### 6.3 M13: Prompt Hooks (`m13_prompt_hooks.rs`, 472 lines)

**Constants:** `MIN_PROMPT_LENGTH = 20`

**Handler:**

#### `handle_user_prompt_submit`
1. Skips short prompts (< 20 chars)
2. Advances breaker tick
3. Periodic blackboard GC (every 720 ticks ~1hr): prunes Complete panes + old tasks after 24h
4. Reads cached field state (r, tick, spheres) from `SharedState`
5. Fetches live thermal from SYNTHEX GET `/v3/thermal` (1000ms)
6. Fetches pending tasks from PV2 GET `/bus/tasks` (1000ms) — breaker-gated
7. Reads blackboard fleet summary (local SQLite, no HTTP)
8. Returns `[FIELD] r=X tick=Y spheres=Z T=W fleet=N/MW` + optional task info

**Private helpers:**
- `read_blackboard_summary(state) -> String` — Compact fleet summary ` fleet=N/MW done=D`
- `parse_field_state(data) -> (r, tick, spheres)` — Test-only, parse PV2 health response
- `parse_temperature(data) -> String` — Parse SYNTHEX temperature
- `parse_pending_tasks(data) -> (count, description, id)` — Parse bus tasks response

**Tests:** 25 (field_state: 5, temperature: 5, pending_tasks: 7, prompt length: 2, message format: 4, blackboard: 2)

### 6.4 M14: Permission Policy (`m14_permission_policy.rs`, 363 lines)

**Types:**

#### `Decision` enum
- `Allow` — Allow the tool call
- `AllowWithNotice` — Allow with informational message
- `Deny` — Deny the tool call

#### `PermissionPolicy` struct
| Field | Type | Default |
|-------|------|---------|
| `always_approve` | `Vec<String>` | `["Read", "Glob", "Grep", "LS", "Agent", "WebSearch", "WebFetch", "TodoRead", "TodoWrite"]` |
| `approve_with_notice` | `Vec<String>` | `["Edit", "Write", "Bash", "NotebookEdit"]` |
| `always_deny` | `Vec<String>` | `[]` (empty) |
| `default_approve` | `bool` | `true` (permissive fleet policy) |

**Evaluation precedence:** deny > approve > notice > default

**Handler:**

#### `handle_permission_request`
1. Extracts `tool_name` from HookEvent (default: `"unknown"`)
2. Evaluates against default `PermissionPolicy`
3. Returns:
   - `Allow` → empty response
   - `AllowWithNotice` → `[ORAC] Auto-approved write operation: {tool_name}`
   - `Deny` → block with `[ORAC] Denied: {tool_name} is blocked by fleet policy`

**Public API:**
- `PermissionPolicy::evaluate(tool_name) -> Decision`
- `PermissionPolicy::add_always_approve(tool)`
- `PermissionPolicy::add_always_deny(tool)`

**Tests:** 25 (default policy: 4, evaluate: 7, add_approve: 2, add_deny: 2, serde: 2, edge cases: 4, comprehensive: 4)
