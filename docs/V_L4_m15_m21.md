### Kuramoto Integration (step_inner)

**Mean-field equation:**
```
dtheta_i/dt = omega_i + (K * receptivity / N) * SUM[ w_ij * sin(theta_j - theta_i) ]
```

- Jacobi method: snapshots old phases, computes new from old (no read-write hazard)
- `k_effective = self.k * self.k_modulation`
- Coupling sum clamped to `[-COUPLING_SUM_CAP, COUPLING_SUM_CAP]` where `COUPLING_SUM_CAP = 3.0`
- Euler step: `new_phase = (d_phase * KURAMOTO_DT + old_phase).rem_euclid(TAU)`
- `KURAMOTO_DT = 0.01`

### Order Parameter

```
r = sqrt[(SUM cos(theta_i) / N)^2 + (SUM sin(theta_i) / N)^2]
psi = atan2(SUM sin(theta_i) / N, SUM cos(theta_i) / N) mod 2pi
```

r in [0, 1]: 0 = incoherent, 1 = perfectly synchronized. Empty network returns default (r=0, psi=0).

### coupling_matrix()

Returns `HashMap<(PaneId, PaneId), f64>` where value = `weight * type_weight` (effective coupling strength).

### VERIFIED: LTP Rule

**Condition:** Both endpoints `PaneStatus::Working`

**Formula:**
```
base_ltp = HEBBIAN_LTP (0.01)

if from.activity_30s > 3 OR to.activity_30s > 3:
    ltp *= HEBBIAN_BURST_MULTIPLIER (3.0)

if from.total_steps < NEWCOMER_STEPS OR to.total_steps < NEWCOMER_STEPS:
    ltp *= HEBBIAN_NEWCOMER_MULTIPLIER (2.0)

new_weight = (weight + ltp).clamp(FLOOR, CEILING)
```

**LTP range:** 0.01 (base) → 0.03 (burst) → 0.06 (burst + newcomer)

### VERIFIED: LTD Rule

**Condition:** Exactly one endpoint `Working`, the other idle

**Formula:**
```
weight_delta = -HEBBIAN_LTD (-0.002)
new_weight = (weight + weight_delta).clamp(FLOOR, CEILING)
```

**Fixed magnitude:** Always -0.002. No boosts for LTD.

### VERIFIED: Idle-Idle Skip

**Condition:** Both endpoints idle → **NO update** (G2 guard)

### VERIFIED: Anti-Saturation Guard (G1)

```rust
if working.len() < 2 { return result; }  // Skip entire STDP cycle
```

Prevents 98%+ weights collapsing to floor when few spheres active.

|----------|-------|--------|
| `HEBBIAN_WEIGHT_FLOOR` | 0.15 | m04_constants |
| `HEBBIAN_SOFT_CEILING` | 0.85 | m18 local constant |
| `HEBBIAN_LTP` | 0.01 | m04_constants |
| `HEBBIAN_LTD` | 0.002 | m04_constants |
| `HEBBIAN_BURST_MULTIPLIER` | 3.0 | m04_constants |
| `HEBBIAN_NEWCOMER_MULTIPLIER` | 2.0 | m04_constants |
| `NEWCOMER_STEPS` | 50 | m04_constants |
| `DEFAULT_WEIGHT` | 0.18 | m04_constants |

### STDP Decision Matrix

| from_working | to_working | Action | Delta |
|:---:|:---:|--------|-------|
| Y | Y | LTP + optional boosts | +0.01 to +0.06 |
| Y | N | LTD | -0.002 |
| N | Y | LTD | -0.002 |
| N | N | SKIP (G2) | 0.0 |

### Iteration Pattern

1. Identify working spheres, apply G1 guard (<2 → skip all)
2. Iterate ALL directed connections
3. Check `opt_out_hebbian` flags on both endpoints
4. Classify as LTP / LTD / skip based on working status
5. Collect updates into Vec (avoids borrow conflict)
6. Apply all updates: `(weight + delta).clamp(FLOOR, CEILING)`
7. Count at-floor connections

### Important Note

This is **state-based Hebbian**, not timing-dependent. No tau parameter, no exponential decay, no delta-t integration. "Spike timing" is the tick interval configured in the main loop (~5 seconds).

### VERIFIED: FSM State Transitions

```
             failure >= threshold
Closed ─────────────────────────────→ Open
  ↑                                     │
  │  half_open_successes >= threshold   │ tick >= open_timeout
  │                                     ↓
  └──────────────────────────────── HalfOpen
                                       │
                  any failure           │
                  Open ←────────────────┘
```

**Closed → Open:** `consecutive_failures >= config.failure_threshold`
- Sets `opened_at_tick = tick`, resets `half_open_successes`, `half_open_requests` to 0

**Open → HalfOpen:** `current_tick - opened_at_tick >= config.open_timeout_ticks`
- Checked via `tick_check()` or `registry.tick_all()`
- Resets `half_open_successes`, `half_open_requests` to 0

**HalfOpen → Closed:** `half_open_successes >= config.success_threshold`
- Resets all counters to 0

**HalfOpen → Open:** Any failure in HalfOpen
- Immediately transitions back to Open for another timeout cycle

## 10. Verification Verdicts

### Target 1: Coupling Register/Deregister — CONFIRMED

- `register()` creates bidirectional connections with `DEFAULT_WEIGHT (0.18)`, seeds frequency with hash diversity, wraps phase to [0, 2pi), rebuilds adjacency index, triggers auto-K if enabled
- `deregister()` removes phases, frequencies, ALL connections involving the sphere, rebuilds index, triggers auto-K
- Both methods maintain structural integrity via `rebuild_index()` post-mutation

### Target 2: STDP LTP/LTD Rules — CONFIRMED

- **LTP:** Both Working → +0.01 base, x3 burst (activity_30s > 3), x2 newcomer (total_steps < 50), max +0.06
- **LTD:** One Working, one idle → -0.002 fixed, no boosts
- **Bounds:** Clamped to [0.15, 0.85] on every update
- **Guards:** G1 (skip all if <2 working), G2 (skip idle-idle pairs)
- **Opt-out:** Per-sphere `opt_out_hebbian` flag respected

### Target 3: Semantic Router Scoring Weights — CONFIRMED

- `DOMAIN_WEIGHT = 0.4` (40%) — phase proximity to target domain
- `HEBBIAN_WEIGHT = 0.35` (35%) — coupling strength from origin
- `AVAILABILITY_WEIGHT = 0.25` (25%) — Idle=1.0, Working=0.6, Blocked/Complete=0.0
- `PREFERRED_BONUS = 0.15` — additive bonus for preferred pane
- Weights sum to exactly 1.0. Test `scoring_weights_sum_to_one()` enforces this.

### Target 4: Circuit Breaker FSM — CONFIRMED

- **3 states:** Closed (default), Open (tripped), HalfOpen (probing)
- **Transitions:** Closed→Open (failure threshold), Open→HalfOpen (tick timeout), HalfOpen→Closed (success threshold), HalfOpen→Open (any failure)
- **3 presets:** Default (5/2/30/1), Aggressive (3/1/15/1), Tolerant (10/3/60/2)
- **Safety bugs fixed:** BUG-M005 (Open→Closed bypass blocked), BUG-L4-003 (no consecutive_failure increment in Open), BUG-L4-006 (opened_at_tick floor = 1)
- **Registry:** Per-pane isolation, optimistic for unknown panes, batch tick processing
