# /acp — Adversarial Convergence Protocol

Launch a 3-round distributed intelligence process. Each round narrows through opposition until only what survives genuine challenge remains.

## Arguments
- `$ARGUMENTS` — The question or decision to analyse

## Protocol

### Round 1: DIVERGENCE
Deploy 3+ specialist background agents, each investigating a different facet. Optionally deploy 9 fleet panes with dimensional roles.

Gate: ALL sources must deliver before Round 2. Use `/loop 5m` to check.

### Round 2: OPPOSITION
Deploy 3+ critic agents. Each attacks a specific R1 specialist's findings. Must be genuinely adversarial — not self-authored FAQ.

9 adversarial fleet roles: **C**ontradiction Finder, **A**lternative Architect, Complexity Audito**R**, **P**riority Challenger, **M**inimalist, **T**est Gap Finder, **H**istorical Pattern Analyst, Consensus **G**atherer, **S**ingle Biggest Risk.

Gate: ALL critics must deliver before Round 3.

### Round 3: VERIFICATION
Deploy verifiers to test the corrected plan against live code and running services. Produce exact line numbers, exact diffs, exact file paths.

Gate: ALL verifiers must deliver before synthesis.

### SYNTHESIS
Only after all 3 gates pass. Synthesize into a single gap-corrected plan. What survives adversarial challenge becomes the plan.

## Rules
1. Gate checks are mandatory — do NOT proceed before gate passes
2. Each round's output feeds the next
3. Critics must be genuinely adversarial
4. Fleet roles are complementary — no duplication
5. The orchestrator designs protocol, not content
6. Wait for delivery — premature synthesis is always worse
